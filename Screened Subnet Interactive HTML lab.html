Interactive Lab: Creating a Screened Subnet (DMZ)

Objective: Configure a screened subnet to place a web server securely, following CompTIA Security+ 1202 objectives.

Scenario

You are the administrator of the fictitious SecureNet XR500 Router. Your company needs to host a public web server. To protect the internal LAN, you must place the server in a DMZ (screened subnet).

Instructions

  1. Log into the router with username admin and password P@ssw0rd!.
  2. Create a new subnet called DMZ with network 192.168.20.0/24.
  3. Assign the web server IP 192.168.20.10 to the DMZ.
  4. Configure firewall rules: Allow HTTP (80) and HTTPS (443) from WAN to DMZ, block LAN access.
