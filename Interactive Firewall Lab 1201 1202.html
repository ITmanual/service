Interactive firewall lab: Block inbound and outbound FTP ports 20 & 21

Use this guided, interactive lab to create Windows Defender Firewall rules that block both inbound and outbound TCP ports 20 and 21 on Windows 11. Follow the clickable steps, view graphics, and run verification commands directly.

Overview diagram

Blocked Inbound FTP (20/21) and Outbound FTP (20/21) are denied at the firewall.

Prerequisites

  • Admin rights: Run steps as an Administrator.
  • Windows 11: UI shown is Windows 11; Windows 10 is similar.
  • Impact: FTP clients/servers will not connect on ports 20/21.

Part 1: Create inbound block rule (ports 20 & 21)

1) Open Windows Defender Firewall with Advanced Security

Start → type Windows Security → open → choose Firewall & network protection → click Advanced settings.

Tip: You can also run wf.msc from Run (Win+R) to open the firewall console directly.

2) Create a new inbound Port rule

Left pane: Inbound Rules → right pane: New Rule… → select Port → Next.

3) Specify ports and block action

Choose TCP → Specific local ports → enter 20,21 → Next → select Block the connection.

4) Apply to profiles and name the rule

Select Domain, Private, Public → Next → name the rule Block Inbound TCP 20-21 (FTP) → Finish.

Part 2: Create outbound block rule (ports 20 & 21)

1) New outbound Port rule

Left pane: Outbound Rules → right pane: New Rule… → select Port → Next.

2) Specify ports and block action

Choose TCP → Specific remote ports → enter 20,21 → Next → select Block the connection.

3) Apply to profiles and name the rule

Select profiles → Next → name the rule Block Outbound TCP 20-21 (FTP) → Finish.

Part 3: Verify rules

GUI verification

Inbound Rules: ensure Block Inbound TCP 20-21 (FTP) is Enabled. Outbound Rules: ensure Block Outbound TCP 20-21 (FTP) is Enabled.

PowerShell verification 
Get-NetFirewallRule -DisplayName "Block Inbound TCP 20-21 (FTP)" | Format-Table DisplayName, Enabled, Direction, Action
Get-NetFirewallRule -DisplayName "Block Outbound TCP 20-21 (FTP)" | Format-Table DisplayName, Enabled, Direction, Action

# Inspect port filters
Get-NetFirewallRule -DisplayName "Block Inbound TCP 20-21 (FTP)" | Get-NetFirewallPortFilter | Format-Table Protocol, LocalPort
Get-NetFirewallRule -DisplayName "Block Outbound TCP 20-21 (FTP)" | Get-NetFirewallPortFilter | Format-Table Protocol, RemotePort
Optional: functional tests 
# PowerShell: test outbound FTP control/data (should fail)
Test-NetConnection -ComputerName ftp.gnu.org -Port 21
Test-NetConnection -ComputerName ftp.gnu.org -Port 20

# If you host an FTP server locally: test inbound
Test-NetConnection -ComputerName localhost -Port 21
Test-NetConnection -ComputerName localhost -Port 20

Part 4: Create rules with PowerShell (alternative)

# Run in elevated PowerShell
New-NetFirewallRule `
  -DisplayName "Block Inbound TCP 20-21 (FTP)" `
  -Direction Inbound `
  -Action Block `
  -Protocol TCP `
  -LocalPort 20,21 `
  -Profile Domain,Private,Public `
  -Enabled True

New-NetFirewallRule `
  -DisplayName "Block Outbound TCP 20-21 (FTP)" `
  -Direction Outbound `
  -Action Block `
  -Protocol TCP `
  -RemotePort 20,21 `
  -Profile Domain,Private,Public `
  -Enabled True

Part 5: Revert or adjust

Disable or enable rules (PowerShell) 
Disable-NetFirewallRule -DisplayName "Block Inbound TCP 20-21 (FTP)"
Disable-NetFirewallRule -DisplayName "Block Outbound TCP 20-21 (FTP)"

Enable-NetFirewallRule -DisplayName "Block Inbound TCP 20-21 (FTP)"
Enable-NetFirewallRule -DisplayName "Block Outbound TCP 20-21 (FTP)"
Remove rules (PowerShell) 
Remove-NetFirewallRule -DisplayName "Block Inbound TCP 20-21 (FTP)"
Remove-NetFirewallRule -DisplayName "Block Outbound TCP 20-21 (FTP)"

Scope & exceptions: Edit a rule → Scope tab to limit IP ranges; Program/Services tabs to constrain to specific executables or services.

End of interactive lab — you have blocked inbound and outbound FTP ports 20 & 21 using Windows Defender Firewall on Windows 11.
