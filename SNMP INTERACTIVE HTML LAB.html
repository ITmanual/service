SNMP Configuration Interactive Lab

CompTIA Network+ N10‑009 – Network Monitoring & Management

Lab Overview

You are the network technician for a small organization. Leadership wants real‑time visibility into device health, interface status, and performance metrics. Your task is to configure SNMP on a router and switch so a monitoring server can collect data and generate alerts.

Network+ N10‑009 Objectives Covered:

  • Monitoring & Metrics (SNMP polling, traps, OIDs)
  • Network Management Protocols (SNMPv1/v2c/v3 differences)
  • Ports & Protocols (UDP 161/162)
  • Security Concepts (authentication, encryption, least privilege)

SNMP is one of the most testable monitoring protocols on the Network+ exam. You’ll see questions about:

  • Which port SNMP uses
  • Difference between polling and traps
  • Why SNMPv3 is preferred for security
  • How community strings work

This lab connects those exam concepts to real device configuration.

Lab Topology

  • SNMP Manager: 192.168.10.50
  • Router R1: 192.168.10.1
  • Switch S1: 192.168.10.2

The SNMP Manager must have IP reachability to both devices.

Step 1 – Review SNMP Concepts

Before configuring, recall these key terms:

  • SNMP Manager: Polls devices and receives traps.
  • SNMP Agent: Runs on the device and exposes OIDs.
  • MIB: Database of objects the manager can query.
  • OIDs: Numeric identifiers for metrics (CPU, interfaces, etc.).
  • Traps: Alerts sent from device to manager.

Q1: Which SNMP component stores OIDs?

A: The SNMP agent using the device’s MIB.

Q2: Which port does SNMP polling use?

A: UDP 161.

Step 2 – Configure SNMPv2c on Router R1

Task: Configure a read‑only community string NETPLUS-RO and basic SNMP identity information.

R1(config)# snmp-server community NETPLUS-RO RO
R1(config)# snmp-server location Main-Closet
R1(config)# snmp-server contact netadmin@example.com
  • Community strings act like passwords in SNMPv2c.
  • “RO” prevents unauthorized configuration changes.
  • Location/contact fields support documentation and monitoring tools.

Step 3 – Configure SNMP Traps

Task: Send traps to the SNMP Manager at 192.168.10.50.

R1(config)# snmp-server enable traps
R1(config)# snmp-server host 192.168.10.50 version 2c NETPLUS-RO
  • Polling = manager asks for data.
  • Traps = device sends alerts automatically.
  • Traps use UDP 162 (common exam question).

Shut down an interface and observe if the manager receives a trap:

R1(config)# interface g0/1
R1(config-if)# shutdown

Step 4 – Configure SNMP on Switch S1

Use the same community string and trap destination for consistency.

S1(config)# snmp-server community NETPLUS-RO RO
S1(config)# snmp-server location IDF-1
S1(config)# snmp-server contact switch-admin@example.com
S1(config)# snmp-server enable traps
S1(config)# snmp-server host 192.168.10.50 version 2c NETPLUS-RO

Monitoring tools expect consistent SNMP configuration across devices. This reduces errors and simplifies management.

Step 5 – SNMPv2c vs SNMPv3 (Security Focus)

SNMPv2c sends community strings in clear text. SNMPv3 adds authentication and encryption.

R1(config)# snmp-server group NETPLUS-GRP v3 priv
R1(config)# snmp-server user netplus-user NETPLUS-GRP v3 \
  auth sha StrongAuthPass1 priv aes 128 StrongPrivPass1
R1(config)# snmp-server host 192.168.10.50 version 3 priv netplus-user
  • SNMPv3 is the correct answer when security is required.
  • Keywords: auth, priv, encryption, integrity.

Step 6 – Verify SNMP Operation

From the SNMP Manager:

snmpwalk -v2c -c NETPLUS-RO 192.168.10.1 system
snmpwalk -v2c -c NETPLUS-RO 192.168.10.2 ifDescr
  • Verification is a required skill on the exam.
  • Common failures: wrong community string, blocked UDP ports, version mismatch.
