CompTIA A+ 220-1202 Lab: Trace Malware to Patient Zero

Use Windows built-ins and disciplined notes to identify the original infected host and its malicious process.

Scenario brief

Four of five PCs and one of three servers are exhibiting malware symptoms (pop-ups, high CPU, suspicious outbound traffic). Your job: determine which host was infected first (“patient zero”), identify the malicious process/persistence, and outline containment.

Objective: Malware detection
Objective: Security & SOHO removal
Objective: Troubleshoot & document
1

Verify symptoms and gather a baseline

Start on two infected PCs and one clean PC. Confirm symptoms are truly malware-related and not misconfiguration or a heavy app. Capture timestamps and commonalities.

- Compare CPU, memory, disk, and network using Task Manager (Ctrl+Shift+Esc) → Processes and Performance.
- Note unusual unsigned executables, random-named processes, or ones spawning repeatedly.
- On the clean PC, collect a “normal” baseline for comparison.
  • Hostnames: Infected vs. clean endpoints.
  • Symptoms: Pop-ups, slowdowns, network spikes, AV alerts.
  • Timestamps: First observed time per host.
  • Process names: Suspicious entries and resource impact.
CompTIA emphasizes verifying symptoms before action, ensuring you don’t misdiagnose normal behavior as malware.
2

Identify malicious processes (Task Manager & Process details)

Use Task Manager and built-in controls to zero in on malicious processes. Cross-check file locations, publisher signatures, and parent-child relationships.

Steps:

  1. Open: Task Manager → Processes; sort by CPU, Network, then Disk.
  2. Inspect: Right-click suspicious process → Open file location and Properties → Digital Signatures.
  3. Track: Go to Details tab; add columns for Command line and Image path (View → Select columns).
  4. Parent: Right-click process → Go to service(s) if applicable; note service associations.

Commands:

wmic process get Name,ProcessId,CommandLine /format:list
tasklist /v
powershell "Get-Process | Select-Object Name,Id,Path,@{n='Cmd';e={$_.Path}}"

Look for executables in %TEMP%, user profile AppData\Roaming, or obscure subfolders under ProgramData.

  • Process name/ID: Across multiple infected hosts.
  • Path: Exact directory; note if nonstandard (e.g., Temp, AppData).
  • Signature: Missing or invalid publisher signature.
  • Parent/child: Spawning patterns indicating persistence or dropper behavior.
3

Locate persistence (Autoruns equivalents, services, scheduled tasks)

Persistence reveals how malware survives reboots and often points to initial delivery. Inspect startup entries, services, and scheduled tasks tied to the suspicious process.

Startup locations:

  • Registry (user): HKCU\Software\Microsoft\Windows\CurrentVersion\Run
  • Registry (machine): HKLM\Software\Microsoft\Windows\CurrentVersion\Run
  • Startup folders: %AppData%\Microsoft\Windows\Start Menu\Programs\Startup
reg query HKCU\Software\Microsoft\Windows\CurrentVersion\Run
reg query HKLM\Software\Microsoft\Windows\CurrentVersion\Run

Services and scheduled tasks:

services.msc
taskschd.msc
  • Unknown service: Random name, relies on suspicious binary path.
  • Scheduled task: Triggers at logon or hourly, executes the malicious program.

Browser & script persistence:

  • Extensions: Check Chromium-based and Edge extensions; remove unknown ones.
  • PowerShell profiles: %UserProfile%\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1
  • Startup scripts: Inspect for .vbs/.js in Startup folders.
4

Correlate events, AV detections, and network indicators

Use Event Viewer and Windows Security logs to build a timeline. Identify the earliest host with detections, process executions, or blocked connections, then verify lateral movement indicators.

Event Viewer:

  • Windows Logs → System: Service starts, driver loads, errors around infection time.
  • Windows Logs → Application: Crashes or unexpected app events tied to suspicious process.
  • Applications and Services Logs → Microsoft → Windows → PowerShell: Suspicious script invocations.

Windows Security (Defender): Virus & threat protection → Protection history; export detections, note first-seen timestamps and file paths.

  • Detection name: Family or heuristic signature.
  • Action: Quarantined/blocked/allowed (user override?).

Network clues:

  • Task Manager → Performance → Open Resource Monitor: Check suspicious outbound connections.
  • netstat: netstat -ano and map PIDs to processes from Step 2.
  • Earliest outbound host: The device with first outbound C2 or distribution behavior is likely patient zero.
5

Build and compare host timelines to find patient zero

Compile first-seen indicators per host (process execution, persistence creation, AV detection, outbound connections). The host with the earliest set of indicators, plus unique artifacts (e.g., installer or dropper), is your patient zero.

  • Unique artifacts: Presence of installer/dropped files not found elsewhere.
  • Propagation role: Shares to others (e.g., SMB write events) or outbound distribution.
6

Containment and documentation (do not remediate yet)

Once patient zero and the malicious process are identified, isolate the host, preserve evidence, and document steps. Defer full cleanup until evidence is captured.

  • Isolate: Disconnect patient zero from the network (disable NIC or remove cable).
  • Snapshot: Copy malicious binary, persistence entries, and logs to secured media.
  • Notify: Record timeline and inform stakeholders per policy.
  • Process details: Name, PID, path, command line, signature info.
  • Persistence: Registry run keys, services, scheduled tasks.
  • Events: System/Application logs, Defender history, netstat mappings.
  • Propagation: Shared folders touched, outbound destinations, timestamps.

Knowledge checks

Use these questions to validate understanding before remediation.

  • Question: Which Windows log category best shows service start anomalies around the infection time?
    • Answer
    Windows Logs → System.
  • Question: Where do common user-level startup entries reside in the registry?
    • Answer
    HKCU\Software\Microsoft\Windows\CurrentVersion\Run.
  • Question: What command maps active connections to process IDs?
    • Answer
    netstat -ano (then correlate PID in Task Manager).

Remediation outline (post-evidence)

After documentation, proceed with: isolate, remove persistence, terminate processes, run AV scan, verify, and re-enable network. Follow organizational policy.

  1. Kill & delete: Stop malicious services/tasks; delete binaries and persistence entries.
  2. AV full scan: Windows Security → Virus & threat protection → Full scan.
  3. Verify: Reboot; confirm no autoruns recreate; netstat clean.
  4. Unblock: Reconnect; monitor for recurrence; patch and credential hygiene.
Document everything—objective mapping, timestamps, artifacts—so your findings support incident response and align to A+ 220-1202 expectations.
