Interactive Lab: Creating a Case in Autopsy (Kali Linux)

You are a cyber investigator beginning a new forensic case. Follow each step, expand the panels, and complete the interactive checks.

Step 1 — Launch Autopsy in Kali Linux+

Open Autopsy using one of the following:

  • Terminal: autopsy
  • Menu: Applications → Forensics → Autopsy

Quick Check: Why should Autopsy be run in a forensic VM instead of the suspect machine?

Step 2 — Open the “Create New Case” Wizard+

Select Create New Case from the Autopsy home screen. You will see fields such as:

  • Case Name
  • Base Directory
  • Description
  • Examiner
  • Case Number (optional)
Step 3 — Case Name+

Purpose: A short, descriptive label for the investigation.

Good Format: Dept-DeviceType-YYYY-MM-DD

Your Turn: Enter a case name for a phishing‑related laptop investigation.

Step 4 — Base Directory+

Purpose: The folder where Autopsy stores all case data.

Recommended: /cases or a dedicated forensic storage path.

Which is the better base directory?

Step 5 — Description+

Purpose: A one‑sentence summary of the incident or reason for analysis.

Your Turn: Write a description for a USB suspected of data exfiltration.

Step 6 — Examiner & Case Number+

Examiner: Your name or analyst ID.

Case Number: Internal tracking ID (optional but recommended).

Which field identifies the person performing the analysis?

Step 7 — Finalize the Case+

Review all fields and click Finish. Autopsy will create the case and prompt you to add a data source.
