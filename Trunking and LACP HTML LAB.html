Scenario & topology

You are the network technician for a small office. The environment uses two managed switches to connect user devices and shared resources.

Your goal is to configure:

  • Trunk ports between switches to carry multiple VLANs.
  • LACP (802.3ad) to bundle two physical links into one logical port-channel.
  • VLAN tagging so a PC on one switch can reach a printer on another switch in the same VLAN.
SW1
Access port: Fa0/10 → PC1 (VLAN 20)
Trunk/LACP: Fa0/1, Fa0/2 → SW2
SW2
Access port: Fa0/11 → Printer1 (VLAN 20)
Trunk/LACP: Fa0/1, Fa0/2 → SW1

CompTIA Network+ relevant objectives

  • 2.1Compare and contrast network topologies and types (switching, VLANs, trunking).
  • 2.3Configure and deploy common Ethernet switching features (VLANs, trunking, LACP).
  • 3.1Use appropriate tools (show commands, verification of VLAN and trunk status).

High-level tasks

  1. Create VLAN 20 on both switches.
  2. Assign access ports to VLAN 20:
    • SW1: Fa0/10 (PC1)
    • SW2: Fa0/11 (Printer1)
  3. Configure trunk ports between SW1 and SW2 on Fa0/1 and Fa0/2.
  4. Bundle the trunk links using LACP into Port-channel1.
  5. Verify that VLAN 20 is allowed across the trunk and that PC1 can reach Printer1 (conceptually).

Concept reminders

VLAN: Logically segments a switch into multiple broadcast domains. Devices in the same VLAN can communicate at Layer 2, even if on different switches, as long as VLAN tagging is consistent and trunks carry that VLAN.

Trunk port: A switch port that carries traffic for multiple VLANs using tagging (e.g., IEEE 802.1Q). Both ends of the trunk must agree on trunking mode and allowed VLANs.

LACP (802.3ad): A standard for bundling multiple physical links into a single logical link (port-channel) for redundancy and increased bandwidth.

Interactive configuration lab

Choose a switch, enter your Cisco-style commands, and click “Check Configuration”. This is a logic check only—no real devices are configured.


Step 1: Create VLAN 20 and name it

Example: vlan 20, name USERS

Step 2: Assign access ports to VLAN 20

SW1: interface Fa0/10 → switchport mode access, switchport access vlan 20
SW2: interface Fa0/11 → same configuration.

Step 3: Configure trunk ports and LACP

On both switches, bundle Fa0/1 and Fa0/2 into Port-channel1 using LACP and allow VLAN 20:

  • interface range fa0/1 - 2
  • channel-group 1 mode active
  • interface port-channel 1
  • switchport mode trunk
  • switchport trunk allowed vlan 20 (or 20,other-vlans)

Step 4: Verification commands (conceptual)

After configuration, you would typically verify with commands such as:

  • show vlan brief – confirm VLAN 20 exists and ports are assigned.
  • show interfaces trunk – confirm trunk status and allowed VLANs.
  • show etherchannel summary – confirm LACP bundle is up.
  • From PC1, ping Printer1 (same VLAN 20, different switch).
