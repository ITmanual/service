Multi‑Protocol Routing Lab: RIP, RIPv2, OSPF, EIGRP & Administrative Distance

In this lab, you will build a small routed network in Cisco Packet Tracer (or similar) and configure RIP, RIPv2, OSPF, and EIGRP. You will then observe how administrative distance (AD) determines which routing protocol “wins” when multiple protocols advertise the same destination network.

Default Administrative Distances (Cisco):
  • Connected: 0
  • Static: 1
  • EIGRP internal: 90
  • OSPF: 110
  • RIP (v1/v2): 120

1. Topology Overview

Create the following logical topology in Packet Tracer:

  • Routers: R1, R2, R3, R4
  • Switches: SW1, SW2, SW3, SW4 (one per router)
  • End Devices: PC1, PC2, PC3, PC4 (one per switch)

1.1 Physical / Logical Layout

  • R1–R2 link
  • R2–R3 link
  • R3–R4 link
  • R4–R1 link (forms a square)
  • Each router connects to its own switch and PC

1.2 Suggested IP Addressing

  • LANs:
    • R1–SW1–PC1: 10.1.1.0/24
    • R2–SW2–PC2: 10.2.2.0/24
    • R3–SW3–PC3: 10.3.3.0/24
    • R4–SW4–PC4: 10.4.4.0/24
  • WAN Links (point‑to‑point /30):
    • R1–R2: 10.12.12.0/30
    • R2–R3: 10.23.23.0/30
    • R3–R4: 10.34.34.0/30
    • R4–R1: 10.41.41.0/30

Task 1 – Build and Address the Topology

  1. Place 4 routers, 4 switches, and 4 PCs in the workspace.
  2. Cable the devices to match the topology described above.
  3. Assign IP addresses to all router interfaces and PCs according to the addressing plan.
  4. Verify basic connectivity on each link using ping between directly connected interfaces.

2. Phase 1 – RIP v1 vs RIPv2

Task 2 – Configure RIP v1 on All Routers

On R1 (adjust networks as appropriate on R2–R4):

R1(config)# router rip
R1(config-router)# version 1
R1(config-router)# network 10.0.0.0
R1(config-router)# no auto-summary
  1. Configure RIP v1 on R1–R4 using the classful network 10.0.0.0.
  2. Use show ip route on each router and note how routes appear (no subnet masks in RIP v1 updates).
  3. Test end‑to‑end connectivity between PCs. Note any issues or odd behavior.

Task 3 – Migrate to RIPv2

On each router:

R1(config)# router rip
R1(config-router)# version 2
R1(config-router)# no auto-summary
R1(config-router)# network 10.0.0.0
  1. Change RIP to version 2 on all routers.
  2. Verify that routes now include subnet masks with show ip route.
  3. Confirm full connectivity between all PCs using ping.
Reflection: What limitation of RIP v1 does RIPv2 solve in this lab’s subnetted environment?

3. Phase 2 – OSPF (SPF) on the Left, RIP on the Right

In this phase, you will run OSPF on the left side (R1–R2) and RIP on the right side (R3–R4), with R2–R3 acting as a redistribution boundary. OSPF uses the Shortest Path First (SPF) algorithm (Dijkstra) to compute the best paths based on cost.

Task 4 – Configure OSPF on R1 and R2

On R1:

R1(config)# router ospf 1
R1(config-router)# router-id 1.1.1.1
R1(config-router)# network 10.1.1.0 0.0.0.255 area 0
R1(config-router)# network 10.12.12.0 0.0.0.3 area 0

On R2:

R2(config)# router ospf 1
R2(config-router)# router-id 2.2.2.2
R2(config-router)# network 10.2.2.0 0.0.0.255 area 0
R2(config-router)# network 10.12.12.0 0.0.0.3 area 0
R2(config-router)# network 10.23.23.0 0.0.0.3 area 0
  1. Ensure OSPF neighbors form between R1 and R2 (show ip ospf neighbor).
  2. Observe OSPF routes in the routing table (show ip route ospf).

Task 5 – Configure RIP v2 on R3 and R4

On R3:

R3(config)# router rip
R3(config-router)# version 2
R3(config-router)# no auto-summary
R3(config-router)# network 10.3.3.0
R3(config-router)# network 10.23.23.0
R3(config-router)# network 10.34.34.0

On R4:

R4(config)# router rip
R4(config-router)# version 2
R4(config-router)# no auto-summary
R4(config-router)# network 10.4.4.0
R4(config-router)# network 10.34.34.0
R4(config-router)# network 10.41.41.0
  1. Verify RIP neighbors and routes on R3 and R4 (show ip route rip).

Task 6 – Redistribute OSPF into RIP and RIP into OSPF

On R2 (OSPF → RIP):

R2(config)# router rip
R2(config-router)# redistribute ospf 1 metric 2

On R3 (RIP → OSPF):

R3(config)# router ospf 1
R3(config-router)# redistribute rip subnets
  1. Check R1’s routing table for networks behind R3/R4 and note the route source (O or R).
  2. Check R4’s routing table for networks behind R1/R2 and note the route source.
Reflection: When both OSPF and RIP advertise the same network, which protocol is chosen and why?

4. Phase 3 – Administrative Distance Experiments

Task 7 – Observe Default Administrative Distance

  1. On R1, identify a network that is learned via both OSPF and RIP (for example, 10.4.4.0/24).
  2. Use show ip route and record which protocol is installed in the routing table.
  3. Explain how the default AD values (OSPF 110, RIP 120) influence this decision.

Task 8 – Change Administrative Distance to Prefer RIP over OSPF

On R1, increase OSPF’s AD so that RIP becomes more preferred:

R1(config)# router ospf 1
R1(config-router)# distance 130
  1. Re‑run show ip route on R1.
  2. Confirm that the route source for the chosen network changes from OSPF (O) to RIP (R).
  3. Describe how this could be used to create primary/backup paths in a real network.

5. Phase 4 – Replace RIP with EIGRP on the Right

Now you will replace RIP on R3–R4 with EIGRP, while keeping OSPF on R1–R2. You will then compare how EIGRP (AD 90) competes with OSPF (AD 110) when both advertise the same routes.

Task 9 – Configure EIGRP on R3 and R4

On R3:

R3(config)# router eigrp 100
R3(config-router)# network 10.3.3.0 0.0.0.255
R3(config-router)# network 10.23.23.0 0.0.0.3
R3(config-router)# network 10.34.34.0 0.0.0.3
R3(config-router)# no auto-summary

On R4:

R4(config)# router eigrp 100
R4(config-router)# network 10.4.4.0 0.0.0.255
R4(config-router)# network 10.34.34.0 0.0.0.3
R4(config-router)# network 10.41.41.0 0.0.0.3
R4(config-router)# no auto-summary
  1. Verify EIGRP neighbors and routes (show ip eigrp neighbors, show ip route eigrp).

Task 10 – Redistribute OSPF ↔ EIGRP

On R2 (OSPF → EIGRP):

R2(config)# router eigrp 100
R2(config-router)# redistribute ospf 1 metric 10000 100 255 1 1500

On R3 (EIGRP → OSPF):

R3(config)# router ospf 1
R3(config-router)# redistribute eigrp 100 subnets
  1. On R1 and R4, inspect the routing table for overlapping routes.
  2. Identify which protocol is preferred when both OSPF and EIGRP advertise the same network.
Reflection: Why does EIGRP usually win over OSPF in this mixed environment? How does administrative distance explain this?

  2. Give a specific example from this lab where administrative distance decided which route was installed.
  3. If this were a real enterprise network, which routing protocol would you standardize on here, and why?
