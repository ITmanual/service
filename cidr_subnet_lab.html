Subnetting and CIDR (Classless Inter-Domain Routing) can be intimidating at first. The transition from pure binary math to real-world network deployment is where many networking students get stuck. To bridge this gap, let's break down the theory using your example: **`192.168.0.1/25`**. After the explanation, you'll find a complete, copy-pasteable, interactive HTML laboratory specifically designed to walk students through this calculation step-by-step. --- ## 1. How CIDR Subnetting Works (The Math) When you see `/25`, it means **the first 25 bits of the 32-bit IP address belong to the Network**. The remaining bits belong to the **Hosts** (individual devices). ### Step A: Finding the Bit Positions We know an IPv4 address has 32 bits divided into 4 octets. Let's look at how `/25` distributes across those octets: $$\begin{aligned} \text{Octet 1:} \quad & 8 \text{ bits} \quad (\text{Cumulative: } 8) \\ \text{Octet 2:} \quad & 8 \text{ bits} \quad (\text{Cumulative: } 16) \\ \text{Octet 3:} \quad & 8 \text{ bits} \quad (\text{Cumulative: } 24) \\ \text{Octet 4:} \quad & \mathbf{1 \text{ bit}} \quad (\text{Cumulative: } 25) \end{aligned}$$ The **4th octet** is our "active" octet because the subnet boundary cuts right through it. * **Network Bits (Leftover in 4th octet):** 1 bit ($25 - 24$) * **Host Bits (Remaining):** 7 bits ($32 - 25$) ### Step B: Calculating the Subnet Mask Since we have 1 network bit in the 4th octet, its binary representation is: $$\mathbf{1}0000000_2 = 128_{10}$$ This makes our Subnet Mask: **`255.255.255.128`**. ### Step C: How Many LANs (Subnets) Can We Create? If we started with a standard Class C network (`/24`), we had 0 network bits in the 4th octet. By moving to `/25`, we "borrowed" **1 bit** from the host portion. $$\text{Number of LANs (Subnets)} = 2^{\text{borrowed bits}} = 2^1 = \mathbf{2 \text{ LANs}}$$ ### Step D: Available IP Addresses per LAN The number of available host addresses in any subnet is determined by the remaining host bits ($h$): $$\text{Total Addresses} = 2^h = 2^7 = 128$$ $$\text{Usable Hosts} = 2^h - 2 = 128 - 2 = \mathbf{126 \text{ usable IPs}}$$ > **Why subtract 2?** Every subnet reserves two special addresses: > 1. **Network Address:** The first address (all host bits set to `0`). Used to identify the subnet itself. > 2. **Broadcast Address:** The last address (all host bits set to `1`). Used to send data to all devices on the subnet. > > ### Step E: Identifying Subnet Ranges Our block size is **128** (which is $256 - 128$ from the mask). Our 2 LANs break down like this: | LAN | Network Address | First Usable IP | Last Usable IP | Broadcast Address | | --- | --- | --- | --- | --- | | **LAN 1** | **`192.168.0.0`** | `192.168.0.1` | `192.168.0.126` | `192.168.0.127` | | **LAN 2** | **`192.168.0.128`** | `192.168.0.129` | `192.168.0.254` | `192.168.0.255` | Since our target IP is **`192.168.0.1`**, it resides on **LAN 1**. Its Subnet Network Address is **`192.168.0.0`**. --- ## 2.
